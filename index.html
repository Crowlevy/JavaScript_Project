Variáveis e Tipos de Dados

Declaração de Variáveis

JavaScript oferece três formas de declarar variáveis:

var

Escopo de função, pode ser redeclarada e atualizada.

var idade = 25;
var idade = 26; // Permitido

let

Escopo de bloco, pode ser atualizada mas não redeclarada.

let nome = "Ana";
nome = "João"; // Permitido
// let nome = "Pedro"; // Erro

const

Escopo de bloco, não pode ser atualizada nem redeclarada.

const PI = 3.14;
// PI = 3.15; // Erro
// const PI = 3.15; // Erro

Operadores em JavaScript

                        
JavaScript possui vários tipos de operadores que permitem realizar diversas operações:

                        
                        

                            

                                
Aritméticos

                                
    
                                    
  • + - Adição
    •

  • / - Divisão
    • 
                                    
  • % - Resto

    • 
                                

                            

                            

                                
Comparação

                                
    
                                    
  • == - Igual a (valor)
    • 
                                    
  • === - Igual a (valor e tipo)
    • 
                                    
  • != - Diferente de (valor)
    • 
                                    
  • !== - Diferente de (valor e tipo)

// Exemplos de operadores
let a = 10, b = 3;

// Aritméticos
console.log(a + b);  // 13
console.log(a - b);  // 7
console.log(a * b);  // 30
console.log(a / b);  // 3.3333...
console.log(a % b);  // 1
console.log(a ** b); // 1000

// Comparação

console.log(a === "10");  // false (compara valor e tipo)

                        Controle de Fluxo
                        
Estruturas de Controle

                        
                        

                            
Estrutura if...else

                            
Executa um bloco de código se uma condição for verdadeira, ou outro bloco se for falsa.

                            

                                
// Exemplo de estrutura if...else
let hora = 14;

if (hora < 12) {
  console.log("Bom dia!");
} else if (hora < 18) {
  console.log("Boa tarde!");
} else {
  console.log("Boa noite!");
}
// Resultado: "Boa tarde!"

                            

                        

                        
                        

                            
Estrutura switch

                            
Avalia uma expressão e executa o código correspondente ao caso que corresponder ao valor.

                            

                                
// Exemplo de estrutura switch
let dia = "quarta";

switch (dia) {
  case "segunda":
    console.log("Início da semana");
    break;
  case "quarta":
    console.log("Meio da semana");
    break;
  case "sexta":
    console.log("Fim da semana!");
    break;
  default:
    console.log("Outro dia da semana");
}
// Resultado: "Meio da semana"

                            

                        

                        
                        

                            
Loops (for, while, do...while)

                            
Loops permitem executar um bloco de código repetidamente.

                            

                                
// Exemplo de loop for
for (let i = 0; i < 3; i++) {
  console.log(`Iteração ${i}`);
}

// Exemplo de loop while
let contador = 0;
while (contador < 3) {
  console.log(`While: ${contador}`);
  contador++;
}

// Exemplo de loop do...while
let j = 0;
do {
  console.log(`Do-while: ${j}`);
  j++;
} while (j < 3);

                            

                        

                        
                        

                            
Dica profissional

                            
Use o operador ternário para condições simples em uma única linha:

                            
let status = idade >= 18 ? "Adulto" : "Menor";

                        

                    

                

            


            
            

                

                    

                        
                    

                    
Funções: O Coração do JavaScript

                

                
                

                    

                        Fundamentos
                        
Declaração e Tipos de Funções

                        
                        
Funções são blocos de código reutilizáveis que executam tarefas específicas e podem retornar valores. No JavaScript moderno, existem várias formas de declarar funções:

                        
                        

                            

                                
// Função tradicional
function somar(a, b) {
  return a + b;
}

// Expressão de função
const multiplicar = function(a, b) {
  return a * b;
};

// Arrow function
const dividir = (a, b) => a / b;

// Função com parâmetros padrão
function saudar(nome = "Visitante") {
  return `Olá, ${nome}!`;
}

console.log(somar(5, 3));       // 8
console.log(multiplicar(4, 2)); // 8
console.log(dividir(10, 2));    // 5
console.log(saudar());          // "Olá, Visitante!"
console.log(saudar("Ana"));     // "Olá, Ana!"

                            

                            
                            

                                
Características das funções em JavaScript:

                                
                                
    
                                    
  • 
                                        
                                        Funções são objetos de primeira classe: podem ser atribuídas a variáveis, passadas como argumentos e retornadas por outras funções.
                                    
    • 
                                    
  • 
                                        
                                        Arrow functions: sintaxe mais concisa e não têm o próprio this, mantendo o escopo externo.
                                    
    • 
                                    
  • 
                                        
                                        Parâmetros padrão: permitem definir valores para parâmetros caso nenhum seja fornecido.
                                    
    • 
                                    
  • 
                                        
                                        Rest parameters: capturam um número indefinido de argumentos como um array.
                                    
    • 
                                

                                
                                

                                    
Quando usar cada tipo?

                                    
    
                                        
  • • Use funções tradicionais quando precisar da palavra-chave this ou do objeto arguments.
    • 
                                        
  • • Use arrow functions para callbacks curtos e quando quiser preservar o contexto do this.
    • 
                                        
  • • Use parâmetros padrão para tornar suas funções mais robustas e flexíveis.
    •

Closures & Callbacks

                        
                        

                            
Closures

                            
Uma closure é uma função que tem acesso a variáveis do escopo externo, mesmo depois que esse escopo é finalizado.

                            

                                
// Exemplo de closure
function criarContador() {
  let contador = 0;
  
  return function() {
    contador++;
    return contador;
  };
}

const meuContador = criarContador();
console.log(meuContador()); // 1
console.log(meuContador()); // 2
console.log(meuContador()); // 3

// A variável 'contador' está
// encapsulada dentro da closure

                            

                        

                        
                        

                            
Higher-Order Functions

                            
São funções que recebem outras funções como argumentos ou retornam funções.

                            

                                
// Exemplo de higher-order function
// map, filter e reduce são exemplos
// de higher-order functions

const numeros = [1, 2, 3, 4, 5];

// map: transforma cada elemento
const dobro = numeros.map(n => n * 2);
// [2, 4, 6, 8, 10]

// filter: seleciona elementos
const pares = numeros.filter(n => n % 2 === 0);
// [2, 4]

// reduce: acumula valores
const soma = numeros.reduce(
  (total, n) => total + n, 0
);
// 15

                            

                        

                        
                        

                            
Aplicações práticas

                            
Funções como map, filter e reduce são essenciais para programação funcional em JavaScript, permitindo código mais limpo e declarativo.

Sobre o Event Bubbling

                                
Eventos em JavaScript "borbulham" dos elementos internos para os externos. Isso significa que um evento em um elemento filho também ativará o mesmo evento em seus elementos pai.

                                

                                    
// Parar a propagação do evento
elemento.addEventListener("click", (e) => {
  e.stopPropagation();
});

                                

                            

                        

                    

                

            


            
            

                

                    

                        
                    

                    
JavaScript Assíncrono

                

                
                

                    

                        Conceitos
                        
Da Callback à Promise ao Async/Await

                        
                        
JavaScript é naturalmente single-threaded, mas permite operações assíncronas para tarefas que podem levar tempo, como solicitações de rede ou operações de arquivo.

                        
                        

                            
Callbacks

                            
A abordagem mais antiga para lidar com operações assíncronas:

                            

                                
// Usando callback para operação assíncrona
function buscarDados(callback) {
  setTimeout(() => {
    const dados = { nome: "João", idade: 30 };
    callback(dados);
  }, 1000);
}

buscarDados((dados) => {
  console.log(dados); // { nome: "João", idade: 30 }
});

                            

                        

                        
                        

                            
Promises

                            
Uma maneira mais elegante de lidar com operações assíncronas:

                            

                                
// Usando Promise para operação assíncrona
function buscarDados() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      const sucesso = true;
      
      if (sucesso) {
        const dados = { nome: "Maria", idade: 25 };
        resolve(dados);
      } else {
        reject("Erro ao buscar dados");
      }
    }, 1000);
  });
}

buscarDados()
  .then(dados => {
    console.log(dados); // { nome: "Maria", idade: 25 }
  })
  .catch(erro => {
    console.error(erro);
  });

                            

                        

                        
                        

                            
Async/Await

                            
Sintaxe moderna que torna o código assíncrono mais legível, parecendo síncrono:

                            

                                
// Usando async/await
async function obterDados() {
  try {
    const dados = await buscarDados();
    console.log(dados); // { nome: "Maria", idade: 25 }
    
    // Pode encadear mais operações assíncronas
    const maisDetalhes = await buscarDetalhes(dados.nome);
    console.log(maisDetalhes);
  } catch (erro) {
    console.error("Ocorreu um erro:", erro);
  }
}

obterDados();

                            

                        

                    

                    
                    

                        Prática
                        
Aplicação Assíncrona com Fetch API

                        
                        
A Fetch API é usada para fazer solicitações HTTP e é baseada em Promises, tornando-a perfeita para operações assíncronas.

                        
                        

                            
// Buscar dados com a Fetch API
fetch('https://api.exemplo.com/dados')
  .then(response => {
    if (!response.ok) {
      throw new Error('Resposta da rede não foi ok');
    }
    return response.json();
  })
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error('Problema na busca:', error);
  });

// A mesma busca com async/await
async function fetchData() {
  try {
    const response = await fetch('https://api.exemplo.com/dados');
    if (!response.ok) {
      throw new Error('Resposta da rede não foi ok');
    }
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Problema na busca:', error);
  }
}

app.get('/usuarios/:id', (req, res) => {
  const id = req.params.id;
  // Buscar usuário do banco de dados

});

app.post('/usuarios', (req, res) => {
  const novoUsuario = req.body;
  // Salvar no banco de dados
  res.status(201).json(novoUsuario);
});

// Iniciando servidor
app.listen(PORT, () => {
  console.log(`Servidor rodando em http://localhost:${PORT}`);
});

Conceitos Avançados

                

                
                

                    

                        Avançado
                        
Hoisting

                        
                        
Hoisting é o comportamento do JavaScript de mover declarações para o topo do escopo atual antes da execução do código.

                        
                        

                            
// Exemplo de hoisting
console.log(x); // undefined (não erro!)
var x = 5;
console.log(x); // 5

// O que realmente acontece:
// var x;        // declaração é "içada"
// console.log(x);
// x = 5;
// console.log(x);

                        

                        
                        

                            
Importante!

                            
Apenas as declarações são içadas, não as inicializações. Com let e const, as declarações também são içadas, mas não são inicializadas, criando uma "Temporal Dead Zone".

                        

                    

                    
                    

                        Avançado
                        
Event Loop

                        
                        

                            
Event Loop

                            
O Event Loop é o mecanismo que permite que o JavaScript execute operações não-bloqueantes, apesar de ser single-threaded.

                            

                                

                                    
                                

                                
                            

                            
O Event Loop constantemente verifica se a Call Stack está vazia. Quando está, ele pega a primeira callback da fila e a coloca na stack para execução.

                        

                    

                    
                    

                        Avançado
                        
Prototypes e Herança

                        
                        
JavaScript usa um sistema de herança baseado em protótipos, não em classes (mesmo com a sintaxe de classe do ES6).

                        
                        

                            
// Objeto construtor
function Animal(nome) {
  this.nome = nome;
}

// Adicionando método ao protótipo
Animal.prototype.falar = function() {
  return `${this.nome} faz algum som.`;
};

// Herança via protótipo
function Cachorro(nome, raca) {
  Animal.call(this, nome); // "super()"
  this.raca = raca;
}

// Conectando protótipos para herança
Cachorro.prototype = Object.create(Animal.prototype);
Cachorro.prototype.constructor = Cachorro;

// Sobrescrevendo o método do protótipo
Cachorro.prototype.falar = function() {
  return `${this.nome} late!`;
};

const rex = new Cachorro('Rex', 'Labrador');
console.log(rex.falar()); // "Rex late!"
